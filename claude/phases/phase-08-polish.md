## 백엔드 구현 항목

### 에러 핸들러 정비
- 모든 예외를 `{ "error": "..." }` 형식으로 통일
- 500 에러: 서버 로그에만 상세 정보, 응답에는 일반 메시지만
- 예상된 에러 (400/401/403/404): 명확한 메시지

### 요청 유효성 검사 강화
- 누락 필드, 형식 오류 → 400
- 최대 길이 초과 → 400

### 프로덕션 CORS
- `ALLOWED_ORIGIN` 환경 변수: 값 있으면 특정 오리진만, 없으면 `*` (개발용)

### 정적 파일 서빙
- 프론트엔드 빌드 결과물을 classpath에 포함
- 비-API 경로의 404 → `index.html` 반환 (SPA 폴백)

## 프론트엔드 구현 항목

### 자동 저장
- isDirty 변경 후 3초 debounce
- 저장 중: 툴바에 "저장 중..." 표시
- 저장 성공: 조용히 완료 (toast 없음, 단 실패 시에는 toast)

### 검색 모달 (Ctrl+K)
- 입력 즉시 결과 (debounce 200ms)
- 검색 대상: 테이블 이름, 컬럼 이름
- 최대 10개 결과
- 결과 없음 상태 표시
- 클릭 → 해당 테이블 노드 선택 + 캔버스 해당 위치로 이동

### 에러 바운더리
- 최상위 컴포넌트 래핑
- 에러 발생 시: "문제가 발생했습니다" + "다시 시도" 버튼
- 에러 정보 콘솔에 기록

### 빈 상태 UX
- 대시보드: 프로젝트 없을 때 일러스트 + "첫 프로젝트 만들기" 버튼
- 버전 목록: 없을 때 "버전이 없습니다. 첫 버전을 저장해보세요"
- 캔버스: 테이블 없을 때 "+ 테이블 추가" 오버레이

### 테마 최종 점검
- 모든 컴포넌트 다크 모드 클래스 확인
- 캔버스 배경, 노드, 엣지 색상 모두 테마 적용

### 빌드 최적화
- 코드 분할 (ERD 라이브러리 별도 청크)
- 빌드 결과물을 백엔드 정적 파일 경로에 출력

## 배포 확인

프로덕션 배포 검증:

```bash
# 1. 프론트엔드 빌드
cd frontend && npm run build

# 2. 백엔드 빌드 (프론트엔드 포함)
./gradlew shadowJar

# 3. 실행
JWT_SECRET=... DB_PATH=./erd.db java -jar build/libs/app.jar

# 4. 확인
curl http://localhost:8080/api/auth/register   # API 동작
open http://localhost:8080                     # SPA 서빙
```

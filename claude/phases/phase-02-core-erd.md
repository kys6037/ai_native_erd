프로젝트 CRUD와 ERD 편집 기능. 저장 후 새로고침해도 복원되어야 한다.

## 백엔드 구현 항목

### ProjectRepository
- `findAll(userId)`: erd_data 제외, 목록용
- `findById(id, userId)`: erd_data 포함
- `create(userId, name, description)`
- `update(id, userId, patch)`: 부분 업데이트
- `delete(id, userId)`
- ErdData ↔ JSON 직렬화/역직렬화
- 모든 쿼리에 userId 조건 포함 (타 사용자 데이터 격리)

### ProjectApi
- 스펙: `04-api-contract.md` 프로젝트 섹션
- 403: 다른 사용자 프로젝트 접근 시 반드시 404가 아닌 403

## 프론트엔드 구현 항목

### ERD 전역 상태 관리
다음 상태를 중앙에서 관리:
- 현재 프로젝트 (ErdData 포함)
- 선택된 테이블 ID
- 선택된 컬럼 이름
- 선택된 관계 ID
- isDirty (미저장 변경 여부)
- 실행 취소 히스토리 (최대 50개 스냅샷)

모든 변경(추가/수정/삭제)은:
1. 상태 업데이트
2. 히스토리에 스냅샷 push
3. isDirty = true

Undo: 이전 스냅샷으로 복원
Redo: 다음 스냅샷으로 복원

### DashboardPage
- 프로젝트 카드 그리드
- 생성, 이름 변경, 삭제
- 카드 클릭 → 프로젝트 열기

### ERD 캔버스
선택한 ERD 라이브러리로 구현. 핵심 동작:

**노드 (테이블)**:
- 헤더: 테이블 이름 + 배경색
- 컬럼 목록: 이름, 타입, PK/FK 배지
- 클릭: 테이블 선택 → 사이드바 편집기 표시
- 드래그: 위치 이동 → 상태 업데이트

**엣지 (관계)**:
- 소스/타겟 핸들 드래그: 관계 생성
- 클릭: 관계 선택
- Cardinality 레이블 표시

**자동 레이아웃**:
- FK 관계 기반 위상 정렬
- 레벨별 행 배치 (BFS 순서)

### 사이드바 편집기
- 테이블 이름, 색상, 스키마 편집
- 컬럼 편집: 타입 변경 시 관련 필드 동적 표시
- FK 설정: 현재 ERD 테이블/컬럼 기반 드롭다운
- Indexes 탭: 인덱스 CRUD

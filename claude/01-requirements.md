# 01 — Project Requirements

## 한 줄 요약
웹 기반 ERD 설계 도구. 사용자가 로그인하여 DB 스키마를 시각적으로 설계하고, DDL을 생성하거나 기존 DB에서 임포트하며, 팀원과 실시간으로 협업한다.

---

## 사용자 스토리

### 인증
- 회원가입/로그인하여 내 프로젝트에 접근한다
- 로그아웃하면 내 데이터에 접근할 수 없다

### 프로젝트 관리
- 여러 ERD 프로젝트를 생성/수정/삭제한다
- 프로젝트 목록에서 원하는 프로젝트를 열 수 있다

### ERD 편집
- 캔버스에 테이블을 추가/이동/삭제한다
- 테이블의 컬럼(이름, 타입, 제약)을 편집한다
- 테이블 간 관계(FK)를 드래그로 연결한다
- 실수를 undo/redo로 되돌린다
- 자동 레이아웃으로 테이블을 정렬한다

### DDL 생성 및 임포트
- 현재 ERD에서 MySQL/PostgreSQL/Oracle/MSSQL DDL을 생성한다
- DDL SQL을 붙여넣어 ERD로 임포트한다
- .sql 파일을 업로드하여 ERD로 임포트한다
- 외부 DB에 연결하여 스키마를 ERD로 가져온다

### 내보내기
- ERD를 PNG 이미지로 내보낸다
- ERD를 PDF 문서로 내보낸다

### 버전 관리
- 현재 상태를 버전으로 저장한다 (커밋 메시지 포함)
- 버전 히스토리에서 두 버전의 차이를 확인한다
- 이전 버전으로 롤백한다
- 두 버전 간 마이그레이션 SQL을 생성한다

### 실시간 협업
- 같은 프로젝트를 여러 사람이 동시에 편집한다
- 다른 사람의 변경사항이 실시간으로 내 화면에 반영된다
- 현재 접속 중인 참여자를 확인한다

### 데이터 사전
- 테이블/컬럼에 비즈니스 설명, 표준, 도메인, 예시를 기록한다

### 인덱스 관리
- 테이블에 인덱스를 추가/수정/삭제한다
- DDL 생성 시 인덱스 구문이 포함된다

### UX
- 다크/라이트 테마를 전환한다
- 테이블/컬럼을 키워드로 검색한다
- 주요 동작에 키보드 단축키를 사용한다 (Ctrl+S, Ctrl+Z 등)

---

## 비기능 요구사항

| 항목 | 기준 |
|------|------|
| 배포 | 단일 실행 파일 또는 컨테이너 |
| DB | 외부 DB 서버 불필요 (임베디드 DB) |
| 인증 | 상태 비저장 토큰 방식 |
| 보안 | SQL Injection, XSS 방지 |
| 협업 | 네트워크 단절 후 재연결 시 자동 복구 |
| 파일 업로드 | 최대 10MB, 텍스트 파일만 |
